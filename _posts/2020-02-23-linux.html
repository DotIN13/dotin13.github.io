开机自动挂载的两种方案

在给系统新增了磁盘以后，如果重启系统我们会发现找不到存储；但是使用fdisk -l可以看到存储空间，说明存储还在。这是因为关机后，挂载已经自动卸载掉了。我们当然可以手动再次将其挂载，但如果每次重启都需要这样手动操作会很不方便。因此我们可以利用自动挂载，这样系统每次开机的时候就可以自动将磁盘挂载上去了。

方案一:修改/etc/fstab文件

用vim或者nano打开/etc/fstab可以看到如下内容：

# /etc/fstab: static file system information.
#
# Use 'blkid' to print the universally unique identifier for a device; this may
# be used with UUID= as a more robust way to name devices that works even if
# disks are added and removed. See fstab(5).
#
# <file system>             <mount point>  <type>  <options>  <dump>  <pass>
UUID=DA18-EBFA                            /boot/efi      vfat    defaults,noatime 0 2
UUID=f8b114c2-73ac-43c0-a4da-dfa8014bf4b2 /              ext4    defaults,noatime,discard 0 1
UUID=9af284d8-4003-44da-9852-13991802f6d8 /home          ext4    defaults,noatime,discard 0 2
UUID=546b5c77-7088-42f4-9653-426a74aea130 swap           swap    defaults,noatime,discard 0 2
tmpfs                                     /tmp           tmpfs   defaults,noatime,mode=1777 0 0

可以很明显的看到文件有6列。
第1列是设备名或者卷标；
第2列是挂载点（也就是挂载目录）；
第3列是所要挂载设备的文件系统或者文件系统类型；
第4列是挂载选项，通常使用defaults就可以；
第5列设置是否使用dump备份，置0为不备份，置1，2为备份，但2的备份重要性比1小；
第6列设置是否开机的时候使用fsck检验所挂载的磁盘，置0为不检验，置1，2为检验，但置2盘比置1的盘晚检验。

本文示例

分别将sda1、sda5、sda6、sdb5挂载到/mnt下相应位置


/dev/sda1                                 /mnt/Stormwind ntfs    defaults         0 0
/dev/sda5                                 /mnt/Karazhan  ntfs    defaults         0 0
/dev/sda6                                 /mnt/Oregma    ntfs    defaults         0 0
/dev/sdb5                                 /mnt/Xander    ntfs    defaults         0 0

将上面的命令添加进fstab后，为了避免可能的错误，我们可以使用mount -a命令来检验编辑的内容是否有错。执行mount -a命令后，用df -h命令查看会发现磁盘已经挂载成功，说明输入没有错误。下次重启的时候系统就可以自动进行挂载了。

注意

  1. 根目录必须优先于其他挂载点
  2. 挂载点必须为已经存在的目录
  3. 卸载时必须保证当前磁盘没有发生读写操作

方案二:修改/etc/rc.d/rc.local文件

将mount命令添加进rc.local文件是另一种实现开机自动挂载的方式

mount /dev/sda1 /mnt/Stormwind
